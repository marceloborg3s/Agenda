// Ele não tem coisas que bases de dados geralmente tem: Validação de dados, confirmação de salvamento do dado(Salvo exatamente como queriamos que fosse).

// Um schema (também conhecido como esquema) é uma representação estruturada de um conjunto de informações ou dados que define a organização, a estrutura e as restrições desses dados. Pode ser pensado como um plano ou um modelo que descreve como os dados devem ser organizados e como as diferentes entidades ou elementos estão relacionados entre si.

// Em diferentes contextos, o termo "schema" pode ter significados ligeiramente diferentes. Aqui estão alguns exemplos:

// Em bancos de dados: Um schema de banco de dados define a estrutura lógica de um banco de dados, incluindo tabelas, colunas, tipos de dados, restrições, chaves primárias, chaves estrangeiras e outras definições relacionadas. Ele especifica como os dados são organizados e armazenados no banco de dados.

// Em linguagens de marcação: Um schema define a estrutura, os elementos permitidos, os atributos e as regras de validação de uma linguagem de marcação específica, como XML (eXtensible Markup Language) ou JSON (JavaScript Object Notation). O schema define a estrutura que os documentos devem seguir para serem considerados válidos.
 
// Em ciência da informação: Um schema é uma estrutura de classificação ou organização de informações que descreve as relações entre conceitos ou entidades. Pode ser usado para representar taxonomias, ontologias ou outros sistemas de classificação.

// Em resumo, um schema é um plano ou uma especificação que define como os dados devem ser organizados e estruturados, seja em um banco de dados, em uma linguagem de marcação ou em outro contexto relacionado à representação e organização de informações.
